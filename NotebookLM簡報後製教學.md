## 前言：為什麼需要後製？

NotebookLM 是 Google 推出的 AI 筆記工具，能夠自動生成簡報。然而，AI 生成的簡報往往需要進一步調整才能符合實際需求：

- **品牌一致性**：需要加入公司 Logo、統一配色
- **內容微調**：修正數據、更新資訊
- **版面優化**：調整排版、提升視覺效果
- **移除浮水印**：去除 NotebookLM 預設的標記

本教學將帶你掌握 NotebookLM 簡報的後製技巧，讓 AI 生成的簡報更加專業。

## 核心問題與挑戰

### NotebookLM 簡報可以修改嗎？

**答案是可以，但需要特殊工具。**

### 為什麼不能直接編輯？

NotebookLM 輸出的 PDF 屬於「**不可編輯的 PDF**」，原因如下：

| 特性 | 說明 |
|-----|------|
| 圖層合併 | 所有元素（文字、背景、圖像）已合併在同一圖層 |
| 本質差異 | 實際上是靜態圖片，而非可編輯的向量文件 |
| 無法選取 | 文字無法直接選取或複製 |

### 解決核心原理

要編輯這類 PDF，主要依賴兩種技術：

1. **OCR（光學字元辨識）**：將圖片中的文字轉換為可編輯文字
2. **AI 圖層分離技術**：將合併的元素重新拆解為獨立物件

## 從源頭優化：素色背景提示詞技巧

> **核心概念**：與其事後費力編輯，不如從生成階段就產出易於編輯的內容

### 為什麼素色背景更容易編輯？

NotebookLM 生成的簡報如果使用漸層、半透明或複雜底紋，後續用圖文工具修改文字時會非常困難。**素色（單一不透明色）背景**的優勢：

| 特性 | 素色背景 | 漸層/複雜背景 |
|-----|---------|--------------|
| 文字修改 | 容易覆蓋重新編輯 | 需精準對齊漸層 |
| 色塊替換 | 一鍵填充 | 需重建漸層效果 |
| OCR 辨識 | 高對比度，辨識準確 | 可能因底色干擾 |
| 圖層分離 | 邊界清晰 | 邊界模糊難分離 |

---

### 素色簡報生成提示詞

當您需要 NotebookLM 生成簡報時，使用以下提示詞：

```
請為我生成一份關於 [您的主題] 的簡報大綱與視覺建議。
在視覺設計上，請採用**『扁平化極簡風格』**。
所有文字後方的裝飾色塊請使用『單一不透明素色』，
嚴禁使用漸層色、半透明效果或複雜底紋，
並確保文字與背景顏色有高對比度，以便於後續進行數位編輯。
```

#### 提示詞關鍵要素解析

| 關鍵詞 | 作用 |
|-------|------|
| 扁平化極簡風格 | 避免 3D 效果、陰影、紋理 |
| 單一不透明素色 | 確保色塊是純色，非漸層 |
| 嚴禁漸層/半透明 | 明確禁止難以編輯的效果 |
| 高對比度 | 提升文字可讀性與 OCR 辨識率 |

---

### 素色圖表/資訊圖表生成提示詞

當您需要生成數據圖表或資訊圖表時：

```
請繪製一張關於 [數據內容] 的圖表。要求如下：

所有的背景請使用**『純白或純色』**。
標題與標籤文字請放在**『素色的矩形色塊』**上方。
避免任何 3D 效果、陰影或紋理感。
視覺呈現需像平面設計手冊一樣乾淨，文字區域必須是單一色值。
```

---

### 素色設計的配色建議

使用高對比度的素色配色方案：

| 用途 | 建議配色 | Hex 色碼範例 |
|-----|---------|-------------|
| 主要內容/結構 | 藍色系 | `#007BFF` |
| 風格定義/代碼 | 紫色系 | `#9B51E0` |
| 融合生成/執行 | 綠色系 | `#28A745` |
| 後製精修/優化 | 橙色系 | `#FD7E14` |
| 主要文字/線條 | 淺灰色 | `#E0E0E0` |
| 背景色/邊框 | 深灰色 | `#121212` |

## 進階方法：YAML 簡報生成標準作業流程

> **核心理念**：將「內容骨架」與「視覺皮膚」分離的模組化設計

這套方法將簡報製作拆解為四個獨立步驟，讓您能夠精確控制 AI 的輸出品質。

---

### 流程概覽

```
┌─────────────────┐    ┌─────────────────┐    ┌─────────────────┐    ┌─────────────────┐
│  1. 內容骨架化  │ → │  2. 風格定義    │ → │  3. 融合生成    │ → │  4. 後製精修    │
│   (Skeleton)    │    │    (Skin)       │    │   (Fusion)      │    │   (Refine)      │
│                 │    │                 │    │                 │    │                 │
│ 產出結構穩定的  │    │ 建立精確的      │    │ 強制 AI         │    │ 突破無法編輯    │
│ 純文字架構檔    │    │ YAML 設計說明書 │    │ 依循規範產出    │    │ 的限制          │
└─────────────────┘    └─────────────────┘    └─────────────────┘    └─────────────────┘
```

---

### 步驟一：內容骨架化 (Skeleton)

**目標**：產出結構穩定的純文字架構檔

#### 操作流程

```
1. 匯入原始資料（PDF、YouTube、文件等）
           ↓
2. 利用提問功能拓展廣度，生成初稿
           ↓
3. 指令 AI 將初稿整理為「頁碼-標題-重點內文」的條列式文檔
           ↓
4. 將整理好的文檔存為獨立來源
   （命名範例：00_簡報架構）
```

#### 輸出格式範例

```
01. Title: 簡報主題
    Points:
    - 重點一
    - 重點二
    - 重點三

02. Title: 第二頁標題
    Points:
    - 說明內容 A
    - 說明內容 B
```

---

### 步驟二：風格定義 (Skin)

**目標**：建立精確的 YAML 設計說明書

#### 兩種取得方式

**方式 A：手動撰寫**

定義全域規範：
- 氛圍、Hex 色碼、字體系統

定義版面規劃：
- 封面頁、圖文頁、數據頁等變體

**方式 B：AI 反推（逆向工程）**

```
1. 上傳喜歡的設計圖至 ChatGPT/Claude
2. 下達指令：「分析此風格並轉寫為 NotebookLM 專用 YAML 代碼」
3. AI 會產出可用的 YAML 風格定義
```

#### YAML 設計說明書結構範例

```yaml
global:
  atmosphere: "專業商務風格"
  colors:
    primary: "#007BFF"
    secondary: "#6C757D"
    accent: "#28A745"
    background: "#FFFFFF"
    text: "#212529"
  fonts:
    title: "Noto Sans TC Bold"
    body: "Noto Sans TC Regular"

layouts:
  cover:
    type: "全版圖片背景"
    title_position: "中央置中"
  content:
    type: "左圖右文"
    image_ratio: "40%"
  data:
    type: "圖表為主"
    chart_style: "扁平化"
```

---

### 步驟三：融合生成 (Fusion)

**目標**：強制 AI 依循規範產出

#### 操作方式

```
1. 在 NotebookLM 中選擇來源：
   - ☑ 簡報架構檔（步驟一產出）
   - ☑ 原始資料

2. 貼入 YAML 代碼（步驟二產出）
   使用 UI Action 或 Prompt Key

3. 要求 AI 嚴格依據架構檔內容，
   並全程套用 YAML 美學框架
```

#### 關鍵提示詞

```
請嚴格依照提供的 YAML 設計規範生成簡報。
內容必須完全遵循架構檔的頁面結構，
視覺風格必須符合 YAML 中定義的配色與版面規則。
```

---

### 步驟四：後製精修 (Refine)

**目標**：突破無法編輯的限制

#### 精修路線圖

```
                    ┌─────────────────────────────┐
                    │        初步簡報              │
                    └──────────┬──────────────────┘
                               │
         ┌─────────────────────┼─────────────────────┐
         ▼                     ▼                     ▼
┌─────────────────┐   ┌─────────────────┐   ┌─────────────────┐
│  微調 YAML       │   │  匯出 PDF       │   │  匯入 Lovart    │
│  關鍵字          │   │  → 轉存 PNG    │   │  /Gamma 進行    │
│                  │   │                 │   │  分層編輯       │
│ 如：極簡改為     │   │                 │   │                 │
│ 建築感           │   │                 │   │                 │
└─────────────────┘   └─────────────────┘   └─────────────────┘
                               │
                               ▼
                    ┌─────────────────────────────┐
                    │         最終成品             │
                    └─────────────────────────────┘
```

#### 步驟三：取得並使用工具

Gemini 會在 Canvas 中生成一個可運行的網頁應用程式，您可以：

1. 直接在 Canvas 中運行測試
2. 下載程式碼部署到自己的伺服器

## 三大實戰應用場景

### 場景一：商務報告的數據更新

**推薦工具**：PDNob

#### 應用情境

公司季度報告需要更新最新數據，但原始簡報由 NotebookLM 生成。

#### 操作要點

1. **精準定位**：利用 OCR 精準定位需修改的數據
2. **統一風格**：一鍵替換字體、顏色為公司標準
3. **品牌識別**：插入統一 Logo 與版權資訊
4. **批量處理**：多份報告同時處理，提升效率

---

### 場景二：學術海報的結構重組

**推薦工具**：Canva Pro 或 Lovart

#### 應用情境

研討會海報需要重新調整版面，提升學術資訊的可讀性。

#### 操作要點

1. **圖標替換**：用圖標替換複雜文字說明，提升可讀性
2. **模塊重組**：分離各模塊，重新規劃整體佈局
3. **重點強調**：為關鍵數據添加醒目背景
4. **視覺層次**：建立清晰的視覺層次結構

---

### 場景三：行銷提案的創意測試

**推薦工具組合**：PDNob + Canva Pro

#### 應用情境

需要製作多個版本的行銷簡報，測試不同創意方向。

#### 操作要點

1. **文案變體**：用 PDNob 生成不同版本的文案
2. **視覺測試**：在 Canva 中搭配不同主視覺、配色、字體
3. **快速迭代**：複製設計頁面，僅替換關鍵元素
4. **A/B 測試**：快速生成多個變體進行測試

└─► 精細圖層編輯 ────────► Lovart

## 常見問題與解決方案

### Q1：轉換成 PPTX 後格式跑版怎麼辦？

**解決方案**：
- 採用「分頁處理」策略
- 僅針對需要修改的頁面進行轉換
- 其餘頁面保持原始 PDF 格式

### Q2：只需要修改一兩頁，有更簡單的方法嗎？

**解決方案**：
- **色塊覆蓋法**：在需要修改的區域覆蓋色塊，再加上新內容
- **PDNob 圖片編輯**：將該頁轉為圖片後直接編輯

### Q3：如何確保格式穩定性？

**解決方案**：
- 選擇原生 PDF 編輯工具（如 PDNob）
- 減少格式轉換次數
- 盡量在同一工具中完成所有編輯

### Q4：免費工具能滿足基本需求嗎？

**解決方案**：
- **輕度使用**：NotebookLM 編輯器的免費額度足夠
- **中度使用**：PDNob 的 20 次免費編輯
- **重度使用**：建議訂閱 Canva Pro 或購買 PDNob 授權

## 總結與建議

### 核心要點回顧

1. **NotebookLM 輸出的 PDF 本質是圖片**，需要特殊工具才能編輯
2. **OCR 和 AI 圖層分離**是解決問題的關鍵技術
3. **選擇合適的工具**比學習所有工具更重要

### 建議的學習順序

```
初學者路線：
NotebookLM 編輯器 → PDNob → Canva Pro

進階用戶路線：
PDNob → Canva Pro → Lovart
```

### 最佳實踐建議

1. **先評估需求**：明確你要修改什麼，再選擇工具
2. **保留原檔**：編輯前務必備份原始檔案
3. **善用免費額度**：先用免費功能測試，確認適合再付費
4. **組合使用**：複雜專案可以組合多個工具的優勢
